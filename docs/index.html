Event

Up 
post /simulation/event
Adds event to session. (simulationEventPost)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body EventReq (optional)
Body Parameter —

Return type

EventRes

Example data

Content-Type: application/json
{
  "glasses" : "440",
  "filter" : "filter",
  "user_id" : "2109fd82-579a-4130-8f9a-ca22f53c4090",
  "action" : "Picked up 440 glasses.",
  "session_id" : "7f6e9b3c-c2e4-495e-8277-67a0893e839d",
  "id" : "e4866c43-08f7-495c-9adb-44c0a1f40f65",
  "timestamp" : "2022-10-16T15:02:32.910Z"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

If successful, returns the JSON object of the added event to the given session id. EventRes

get /simulation/session
Retrieve session record with optional fields such as events, participants and scene. (simulationSessionGet)

Query parameters

id (optional)
Query Parameter — Unique identifier to find record. If left empty, will return all the session records.
participants (optional)
Query Parameter — Optional flag to retrieve participants linked to session.
events (optional)
Query Parameter — Optional flag to retrieve events linked to session.
scene (optional)
Query Parameter — Optional flag to retrieve scene linked to session.

Return type

inline_response_200

Example data

Content-Type: application/json
{
  "date" : "2022-10-11T14:42:38.817Z",
  "duration" : "duration",
  "description" : "Dummy session description",
  "id" : "9fdf0c31-1493-4a20-bc79-f3e8f1c7e1b2",
  "events" : [ ],
  "participants" : [ ],
  "scene" : [ {
    "id" : "d00fb22e-a352-4554-849e-929d957fe24c",
    "evidences" : [ ]
  }, {
    "id" : "d00fb22e-a352-4554-849e-929d957fe24c",
    "evidences" : [ ]
  } ]
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Returns the session record with optional participants, events and scene field. inline_response_200

post /simulation/session
Create new session record that can be linked with other records. (simulationSessionPost)

Query parameters

description (optional)
Query Parameter — Description field is used to contextualise the session. What is the purpose of this session, who is participating, etc...

Return type

inline_response_200_1

Example data

Content-Type: application/json
{
  "date" : "2022-10-11T14:42:38.817Z",
  "duration" : "duration",
  "description" : "Dummy session description",
  "id" : "7f6e9b3c-c2e4-495e-8277-67a0893e839d"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Returns the newly created session record that can now be linked to participants, a scene with evidence and events. inline_response_200_1
